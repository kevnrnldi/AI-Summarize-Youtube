 Peta Jalan Implementasi Backend (Tanpa Kode)
 🏗️ Fase 1: Inisialisasi & Fondasi ProjekLangkah awal untuk menyiapkan medan tempur di komputermu.[ Done]
       Inisialisasi NPM: Membuat file package.json baru di dalam folder projek.
   [Done] Instalasi Modul Utama: Mengunduh pustaka wajib: express, mongoose, dotenv, dan cors.
   [Done] Instalasi Modul Spesifik: Mengunduh pustaka youtube-transcript (untuk mengambil teks video) dan SDK resmi AI (misalnya @google/generative-ai untuk Gemini).
   [Done] Konfigurasi Keamanan: Membuat file .env untuk menyimpan port dan token AI, serta membuat file .gitignore agar file rahasia 
       tersebut tidak terunggah ke internet.
   [Done] Arsitektur Folder: Menyusun folder khas Express seperti kemarin: config/, models/, controllers/, dan routes/.


🗄️ Fase 2: Perancangan Skema Database (Mongoose)Merancang bentuk brankas di MongoDB untuk menampung data ringkasan dan obrolan interaktif.
   [Done] Skema Analisis Video Utama: Membuat model yang mencatat ID unik YouTube, judul video, teks transkrip utuh, dan teks hasil ringkasan dari AI.
   [Done] Skema Sub-Dokumen Chat (Array): Di dalam model video tersebut, sisipkan sebuah struktur array untuk menampung riwayat obrolan. 
       Setiap objek di dalam array ini harus mencatat: siapa yang berbicara (user atau ai) dan apa isi pesannya.


🧩 Fase 3: Pembuatan Fungsi Alat Bantu (Core Utilities)Membuat fungsi-fungsi kecil terisolasi yang bertugas menyelesaikan satu masalah spesifik.
   [Done] Ekstraktor ID YouTube: Membuat fungsi logika (bisa pakai pencocokan teks/Regex) untuk memotong URL 
       YouTube panjang dari pengguna menjadi hanya 11 karakter ID uniknya saja (contoh: https://youtu.be/dZ_n9ic1KrM diperas menjadi dZ_n9ic1KrM).   
   [Done] Penyedot Teks (Scraper Utility): Membuat fungsi yang menerima ID YouTube, memicu pustaka youtube-transcript, lalu menggabungkan pecahan-pecahan teks 
       berdurasi detik menjadi satu kesatuan teks paragraf yang utuh dan bersih.


🧠 Fase 4: Integrasi Servis Otak AIMenghubungkan Express kamu ke server kecerdasan buatan luar.
   [Done] Koneksi Kunci AI: Membuat file konfigurasi yang memvalidasi apakah token API di file .env sudah terbaca dengan benar oleh sistem.
   [Done] Perancangan Instruksi (Prompt Engineering): Menyusun teks perintah baku yang akan dikirim ke AI. Instruksi harus memerintahkan AI 
       untuk meringkas transkrip video ke dalam format poin-poin yang rapi.


🚏 Fase 5: Routing & Controller Utama (Fitur Analisis)Membangun pintu masuk dan logika untuk fitur pertama: Masukkan Link $\rightarrow$ Muncul Ringkasan.
   [Done] Jalur Pintu Masuk (Route): Membuat rute POST /api/analyze.
   [Done] Logika Pemeriksaan (Controller - Bagian 1): Di dalam controller, buat logika untuk mengecek ke database MongoDB: "Apakah ID video ini sudah pernah dianalisis?"
   [Done] Logika Eksekusi Baru (Controller - Bagian 2): Jika belum ada di database, picu Fungsi Alat Bantu (Fase 3) untuk ambil teks, tembak ke Servis AI (Fase 4)
       untuk diringkas, lalu simpan hasil utuhnya ke MongoDB.
   [Done] Logika Respons Akhir: Mengirimkan data hasil analisis (baik hasil ambil dari DB atau hasil tembakan AI baru) kembali ke pengguna dengan status 200 OK.
   

💬 Fase 6: Logika Chatbot Kontekstual (Fitur Diskusi)Membangun fitur paling seru: Tanya jawab seputar isi video yang sedang dibuka.
   [Done] Jalur Pintu Masuk Chat: Membuat rute POST /api/chat/:videoId.[Done] Pengumpul Konteks (Context Gathering): Saat pengguna mengirim pertanyaan baru, 
       controller harus mengambil transkrip video tersebut dari MongoDB berdasarkan :videoId yang sedang aktif.
   [Done] Penyusun Memori Jangka Pendek: Mengambil maksimal 5 sampai 10 riwayat chat terakhir dari database untuk dijadikan modal ingatan AI.
   [Done] Formulasi Tembakan Chat ke AI: Mengirimkan paket komplit ke AI: (Transkrip Video + Riwayat Chat Lama + Pertanyaan Baru).
   [Done] Pencatatan Sejarah: Setelah AI menjawab, simpan pertanyaan pengguna dan jawaban AI tersebut ke dalam array riwayat chat di MongoDB, 
       lalu tampilkan hasilnya di layar pengguna.


🧪 Fase 7: Pengujian Kasus Ekstrem (Testing via Postman)Menjadi "satpam kejam" untuk menguji ketangguhan kodinganmu sendiri.
   [Done] Uji Coba Duplikasi Tab: Masukkan link video A berulang kali di Postman, pastikan di tembakan kedua prosesnya instan karena mengambil dari database,
       bukan menembak AI lagi.
   [Done] Uji Coba Lupa Link: Kirim request kosong tanpa link, pastikan Express menangkapnya dan mengeluarkan error 400 Bad Request.
   [Done] Uji Coba Chat Lanjutan: Lakukan chat di video A, lalu pindah ke video B. Pastikan saat kamu bertanya tentang video B, AI tidak menjawab
       menggunakan data dari video A.
